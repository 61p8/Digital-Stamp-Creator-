## แอปในชุดนี้

### App 1 · ตราชื่อ-ลายเซ็น · Signature Stamp
ไฟล์: [`Sign_Stamp_Creater.html`](./Sign_Stamp_Creater.html)

สร้างตราชื่อ ตราลายเซ็น หรือตราตำแหน่ง พร้อมข้อความ 3 บรรทัด เลือกฟอนต์/สี/รูปร่างได้อิสระ

**เหมาะกับ:** ตรารับรองเอกสาร · ตราชื่อพนักงาน · ตราหน่วยงาน · ตราสำหรับเซ็นรับ

- รูปร่าง 8 แบบ: วงกลม, วงกลมเส้นเดียว, สี่เหลี่ยม, สี่เหลี่ยมเส้นเดียว, สี่เหลี่ยมมุมมน (2 แบบ), วงรี, ไม่มีขอบ
- ข้อความ 3 บรรทัด (บน/กลาง/ล่าง) ตั้งค่าฟอนต์, ขนาด, สไตล์ B/I/U อิสระ
- โหลดฟอนต์จากเครื่องได้ (Chrome/Edge)
- เลือกสี: 12 preset + color picker + hex code
- ลากเส้นแบ่งใน preview เพื่อปรับสัดส่วน · ลากข้อความขึ้น-ลงเพื่อจัดตำแหน่ง
- บันทึก PNG / SVG / คัดลอกรูป / พิมพ์ / บันทึกค่า .json
- **บันทึกอัตโนมัติ** ใน browser

### App 2 · ตรารับรองเอกสาร · Document Stamp
ไฟล์: [`Other_Stamp_Creater.html`](./Other_Stamp_Creater.html)

ตรายางสี่เหลี่ยมแบบองค์กรสำหรับรับรอง/อนุมัติเอกสาร พร้อมช่อง Reason / Date / Approved / Checked / Writer

**เหมาะกับ:** เอกสาร PCR · ใบอนุมัติ · ใบยกเลิก · ใบรับรอง · เอกสารควบคุมเอกสาร (DCC)

- หัวตราข้อความปรับได้ (เช่น ABOLISHED, APPROVED, REJECTED, RELEASED)
- ช่อง Reason **เพิ่ม/ลบบรรทัดได้** ตามต้องการ
- รูปแบบวันที่ 11 แบบ (ค.ศ., พ.ศ., วัน-เดือน-ปี, ภาษาญี่ปุ่น ฯลฯ)
- ช่อง Approved / Checked / Writer (ปรับความกว้างได้)
- ลากขอบ/มุมกรอบเพื่อปรับขนาดตรา · ลากเส้นประใน preview เพื่อเลื่อนตำแหน่ง
- ปรับความนุ่มของเส้นแบบตรายาง
- รองรับ 3 ภาษา: ไทย / English / 日本語
- บันทึก PNG / SVG / คัดลอกรูป
- **บันทึกอัตโนมัติ** ใน browser

## วิธีใช้

### แบบที่ 1: เปิดในเครื่อง
ดับเบิลคลิก `index.html` → เลือกแอปที่ต้องการ → ใช้ได้ทันที

### แบบที่ 2: Host บน GitHub Pages (แชร์ในทีม)
1. สร้าง repo ใหม่บน GitHub (Public)
2. Upload ไฟล์ทั้งหมด (`index.html`, `Sign_Stamp_Creater.html`, `Other_Stamp_Creater.html`) เข้า repo
3. ไปที่ **Settings** → **Pages**
4. เลือก **Source: Deploy from a branch**
5. เลือก **Branch: main** / **Folder: / (root)** → กด Save
6. รอ 1-2 นาที จะได้ URL `https://[ชื่อผู้ใช้].github.io/[ชื่อ-repo]/`
7. ส่ง URL ให้คนในทีมเปิดหน้าหลักได้เลย

## เรื่องการบันทึกค่า (สำคัญ)

ข้อมูลทั้งหมดเก็บอยู่ในเครื่องผู้ใช้ **ไม่มีการส่งข้อมูลไปไหน**

- **บันทึกอัตโนมัติ**: ทุกการเปลี่ยนแปลงเซฟลง browser localStorage ทันที
- **แยกกันสมบูรณ์**: คุณ A กับคุณ B เปิด URL เดียวกัน — เห็นค่าของตัวเองคนละหน้า ไม่ปนกัน
- **อยู่กับ browser**: เปลี่ยนเครื่อง / เปลี่ยน browser = เริ่มใหม่
- **ปุ่ม "ล้าง"** (App 1): ลบค่าที่บันทึกไว้ทั้งหมด กลับเป็นค่าเริ่มต้น
- **ปุ่ม "บันทึกค่า"** (App 1): download ไฟล์ .json ไว้ backup หรือแชร์ template
